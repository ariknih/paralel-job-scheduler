Core Architecture

The benchmarking application distributes a large workload of retail records (`OnlineRetail.csv`) using a scheduler and measures the throughput across three key stages:

  • SISD (Single Instruction Single Data): Runs a single-threaded sequential loop (`sisd.py`) to process the transactions.
  • MIMD (Multiple Instruction Multiple Data): Disperses data chunks into multiple concurrent thread workers (`mimd.py`) utilizing Python's ThreadPoolExecutor.
  • Load Balancing Strategy: Regulates how workloads are assigned via Equal, Round Robin, or Weighted strategies (`load_balancer.py`).

Standard Library Implementation

The system relies entirely on standard Python modules (no external pip dependencies) to ensure high portability and pure algorithmic measurement.

Load Balancing Strategies

Load balancing dictates how transactions are split among processing threads. Our scheduler implements three models:

⚖️

Equal Distribution

Splits the dataset into contiguous, identical block slices. Simple, but susceptible to load imbalance if records have varying processing times.

🔄

Round Robin

Distributes lines one by one cyclically (e.g. Worker 0 takes line 0, Worker 1 takes line 1, and so on). Keeps record counts highly balanced.

📈

Weighted Distribution

Assigns records based on custom worker performance weights, letting faster processor threads handle larger proportions of the workload.

Developer & Run Guide

Follow these commands to generate the workload dataset and execute the benchmark program:

1. Generate OnlineRetail.csv Mock Dataset

python generate_dataset.py

2. Run Benchmark with Equal Load Balancer

python main.py --workers 4 --strategy equal

3. Run Benchmark with Weighted Distribution

python main.py --workers 4 --strategy weighted --weights 4,2,1,1
